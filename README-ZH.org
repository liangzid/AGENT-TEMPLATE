* 这是什么？

AGENT-TEMPLATE 是一个可复用的 AI 编程智能体（coding agent）配置模板。它提供了一套经过实战检验的 agent 指令（prompts）和权限配置，使得 Claude Code / pi 等 coding agent 工具能够以规范、高效、安全的方式协助你进行软件开发与学术写作。

推荐agent： PI Coding Agent.

本仓库的核心理念是：*prompt 的可复用性比单次产出的代码更有价值*。与其每次临时调整 agent 的行为，不如维护一套稳定、通用的配置模板。

* AGENTS.md 的核心设计

** 对话规则（Rules）

AGENTS.md 中的规则优先级高于其他内容。核心要点：

1. *允许并鼓励 agent 提出异议*。若用户的假设或前提有误，agent 必须明确指出，而非一味迎合。
2. *禁止捏造*。不确定时，通过读文件、跑命令或搜索来获取信息。若仍然无法确定，直说 "No evidence to show..." 或 "I do not know that."
3. *遇到困惑立即停下*。若用户指令不清或存在歧义，必须追问讨论，不可沉默推进。
4. *最小化代码改动*。不允许进行与当前任务无关的重构、格式化或其他旁路修改。
5. *Grilling（追问式规划）*。在制定计划时，agent 需要对用户的每个决策分支逐一追问，直到所有依赖关系被充分讨论。每一步只能提一个问题。
6. *记录讨论*。每 5 轮对话后，将关键讨论点简要记录至 =./records/[Task].org=。记录需剔除噪声与错误探索，只保留必要结论。

** 编码守则

- 禁止 self-commit（=git commit=）。由用户决定何时提交。
- 单个函数长度不可过长或过大。
- *禁止 mock 数据*，即使处于调试阶段。
- 命名以清晰为第一原则。例如 =maio2025.py= → =loading_miao2025.py=。
- Python 项目使用 =uv= 管理。

** 代码审查要求

每个代码文件必须包含以下格式的文件头：

#+begin_example
======================================================================
`(upcase (file-name-nondirectory (file-name-sans-extension (buffer-file-name))))`

1. (What-this-file-aims-to-do)
2. (The-calling-chains-or-graphs-of-each-function-or-components)
3. (Modification-history-and-reason)

    Author: Zi Liang <zi1415926.liang@connect.polyu.hk>
    Copyright © `(format-time-string "%Y")`, Zi Liang, all rights reserved.
    Created: `(format-time-string "%e %B %Y")`
======================================================================
#+end_example

此外，每个函数及关键位置都需要注释，并显式标注需要用户审查的关键点。

** 论文写作守则

- 始终使用 LaTeX 格式，不写 markdown。
- 数学公式用 =$$= 或 =\begin{...}\end{...}=，禁止使用=\(\=)。
- 禁止使用：solely、leverage、delve into、tapestry、furthermore 等词汇。
- 禁止使用 em dash（—）、double em dash（--），以及分号（;）。
- 禁止在 enumerate 等环境中加粗第一句话（= \textbf{} =）。
- 禁止缩写（如 it's → it is），减少使用 ='s= 结构。
- 注意上下文已有的缩写，不要重复展开。
- 保留团长（用户）所做的任何格式调整。

* 设计哲学

- *少聊天，多发文档*。
- No SDD
- 人类一定要多读文档，多review, 多询问。
- *prompt 语言不重要*。英文最方便，但是不忌讳使用中文。用自己最顺手的方式写。
- *可复用性第一*。一个好的 prompt 模板可以服务无数个任务，它的价值远超单次产出的代码。
